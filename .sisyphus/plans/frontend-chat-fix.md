# Frontend Chat Interface & Backend Session Management Fix

## TL;DR

> **Quick Summary**: Fix broken chat interface by replacing dashboard with ChatGPT-like UI, implement session management with external_session_id (UUIDv7), and integrate Claude SDK session tracking with polling/streaming support.
>
> **Deliverables**:
> - ChatGPT-like chat interface with message input and streaming display
> - Working session management with no-dialog creation flow
> - Backend support for external_session_id and Claude SDK session_id
> - Polling (2s default) with streaming toggle
> - Full-stack test coverage
>
> **Estimated Effort**: Large
> **Parallel Execution**: YES - 4 waves
> **Critical Path**: DB Migration → Backend Session Changes → Frontend Hook → Chat UI Integration

## Context

### Original Request
User reported multiple frontend issues:
1. Project creation: Worktree Path required but should be optional
2. Sidebar: Workspace list not displaying correctly
3. No chat interface: No input box to interact with backend
4. Session management: Dialog-based creation problematic, needs external_session_id support
5. Data flow: Need polling (default 2s) or streaming with incremental message display

### Interview Summary
**Key Discussions**:
- Worktree Display: User wants Workspace list in sidebar (not Worktree itself)
- Session Creation: No dialog, direct creation with auto-generated title "New Chat"
- Polling Interval: 2 seconds (default), with streaming toggle
- Test Strategy: Full-stack tests (Vitest + pytest + Agent QA)
- Database: Cleanup for this plan only (one-time)

**Research Findings**:
- UUIDv7: Use `uuidv7` npm package for time-sortable IDs
- SSE streaming: Already exists in backend (`/messages/send`)
- InputArea: Component already exists at `/web/components/InputArea.tsx`
- Sessions hook: Missing, needs to follow `useProjects.ts` pattern

### Metis Review
**Identified Gaps** (addressed):
- `external_session_id` purpose: Frontend generates for correlation, stored alongside SDK session_id
- InputArea exists: Will integrate existing component, not recreate
- SDK session_id capture: Add backend task to capture and return in `send_ai_prompt`
- Empty state: Show "Select workspace to start" when no workspace selected
- Session title: Use "New Chat" as default
