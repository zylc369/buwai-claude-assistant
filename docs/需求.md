## 目录说明

1. server目录里面作为服务端，使用python技术栈。
2. web目录里面作为前端，与服务端交互。

## 服务端

当前服务端还是一个命令行工具，我需要将它升级成web server，暴露若干API。服务端支持同时服务多个工作目录，和claude的工作目录的概念一致。




### 数据存储

使用数据库表存储数据。创建一个数据适配层，当前使用sqlite存储数据，但是要比较容易的替换为其他数据库（如：mysql、云端数据库）。

#### 表关系图

```
project
   │
   ├─── 1:N ──→ session ──→ 1:N ──→ message
   │
   └─── 1:N ──→ workspace
```

####  project 表

项目表，存储项目基本信息。

| 字段名              | 类型          | 约束        | 说明             |
| ------------------- | ------------- | ----------- | ---------------- |
| `id`                | unsigned long | PRIMARY KEY | 表ID，自增       |
| `project_unique_id` | text          | UNIQUE      | 项目唯一标识     |
| `worktree`          | text          | NOT NULL    | 工作树路径       |
| `branch`              | text        | -                         | Git 分支名称          |
| `name`              | text          | -           | 项目名称         |
| `time_initialized`  | unsigned long | -           | 初始化时间戳     |
| `time_created`      | unsigned long | NOT NULL    | 创建时间         |
| `time_updated`      | unsigned long | NOT NULL    | 更新时间         |

---

#### session 表

会话表，存储对话会话信息。

| 字段名                | 类型          | 约束                                         | 说明                  |
| --------------------- | ------------- | -------------------------------------------- | --------------------- |
| `id`                  | unsigned long | PRIMARY KEY                                  | 表ID，自增            |
| `session_unique_id`   | text          | UNIQUE                                       | 会话唯一标识          |
| `project_unique_id`   | text          | NOT NULL, FK → project.project_unique_id     | 所属项目 ID，级联删除 |
| `workspace_unique_id` | text          | NOT NULL, FK → workspace.workspace_unique_id | 工作空间 ID           |
| `directory`           | text          | NOT NULL                                     | 工作目录              |
| `title`               | text          | NOT NULL                                     | 会话标题              |
| `time_created`        | unsigned long | NOT NULL                                     | 创建时间              |
| `time_updated`        | unsigned long | NOT NULL                                     | 更新时间              |
| `time_compacting`     | unsigned long | -                                            | 压缩时间戳            |
| `time_archived`       | unsigned long | -                                            | 归档时间戳            |

**索引：**

- `session_project_unique_idx` on `project_unique_id`
- `session_workspace_unique_idx` on `workspace_unique_id`

---

####  message 表

消息表，存储对话中的消息元信息。

| 字段名              | 类型          | 约束                                     | 说明                                  |
| ------------------- | ------------- | ---------------------------------------- | ------------------------------------- |
| `id`                | unsigned long | PRIMARY KEY                              | 表ID，自增                            |
| `message_unique_id` | text          | UNIQUE                                   | 消息唯一标识                          |
| `session_unique_id` | text          | NOT NULL, FK → session.session_unique_id | 所属会话 ID，级联删除                 |
| `time_created`      | unsigned long | NOT NULL                                 | 创建时间                              |
| `time_updated`      | unsigned long | NOT NULL                                 | 更新时间                              |
| `data`              | text (JSON)   | NOT NULL                                 | 消息详细数据，类型为 `MessageV2.Info` |

**索引：**

- `session_session_unique_idx` on `session_unique_id`



#### workspace 表

工作空间表，存储项目的工作空间信息。

| 字段名                | 类型          | 约束                      | 说明                  |
| --------------------- | ------------- | ------------------------- | --------------------- |
| `id`                  | unsigned long | PRIMARY KEY               | 表ID，自增            |
| `workspace_unique_id` | text          | UNIQUE                    | 工作空间唯一标识      |
| `branch`              | text          | -                         | Git 分支名称          |
| `name`                | text          | -                         | 工作空间名称          |
| `directory`           | text          | -                         | 工作目录              |
| `extra`               | text (JSON)   | -                         | 额外信息              |
| `project_unique_id`   | unsigned long | NOT NULL, FK → project.id | 所属项目 ID，级联删除 |

**索引：**

- `session_project_unique_idx` on `project_unique_id`



### 支持的API

1. 项目数据查询：
   1. 支持分页查询获得项目信息列表。
   2. 根据项目名、项目唯一ID、项目ID获取候选的项目列表。项目名允许模糊查询。
2. session数据查询：
   1. 支持分页查询获得session数据，项目唯一ID和workspace唯一ID必传。
   2. 根据session唯一ID获得session数据。
3. message数据查询：
   1. 支持分页查询获得message数据，项目唯一ID、workspace唯一ID、session唯一ID必传。
   2. 根据message唯一ID获得message数据。
4. workspace数据查询：
   1. 支持分页查询获得workspace列表，项目唯一ID必传。
   2. 根据workspace名、workspace唯一ID、workspace ID获取候选的项目列表。workspace名允许模糊查询。
5. 接收提示词，调用AI的接口，项目唯一ID、workspace唯一ID、session唯一ID必传。
6. 前端长连接订阅session的流式输出的API，项目唯一ID、workspace唯一ID、session唯一ID必传。



### 前端

1. 顶部切换和新增项目
2. 侧边栏切换项目下的工作空间，点击某个工作空间，侧边栏弹出显示：工作空间名字、新建会话按钮、会话列表（不显示已归档的会话）。
3. 新建会话按钮，点击后新建一个会话，如果当前会话没有任何消息，不要再新建会话。
4. 点击会话列表要切换会话，即下次请求的时候要使用被切换的会话唯一ID。
5. 中间是消息展示区域
6. 底部是提示词输入区域，需要有输入框和发送按钮。
   1. 快捷键：command+enter、shift+enter换行，enter发送。
7. 页面打开打开的时候：
   1. 先请求项目列表。
   2. 然后搜索浏览器本地存储内是否有上次打开的`项目唯一ID + workspace唯一ID + session唯一ID`。如果有，默认切换到上次打开的项目，然后请求项目的workspace并切换到上次打开的workspace，然后请求workspace的session列表并切换到上次打开的session。
